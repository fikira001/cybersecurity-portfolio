# SQL Query Filtering for Security Investigations

## Overview

This project demonstrates how SQL can be used to investigate security events by filtering and retrieving relevant records from organizational databases. The activity focuses on identifying suspicious login attempts and locating employee assets requiring security updates.

## Scenario

As a security analyst, I investigated login activity and employee records using SQL queries. By applying conditional filtering, I identified failed login attempts, suspicious activity, and employee systems requiring administrative action.

## Skills Demonstrated

- SQL Querying
- Security Log Analysis
- Database Filtering
- Incident Investigation
- Access Monitoring
- Data Analysis

## SQL Concepts Used

- `SELECT`
- `FROM`
- `WHERE`
- `AND`
- `OR`
- `NOT`
- `LIKE`
- Wildcards (`%`)

## Investigation Tasks

- Retrieved failed login attempts after business hours.
- Identified login attempts on specific dates.
- Investigated login attempts originating outside Mexico.
- Retrieved Marketing department employees in the East building.
- Retrieved employees from the Finance and Sales departments.
- Identified employees outside the Information Technology department.

## Key Takeaways

- SQL is an essential skill for investigating security events stored in databases.
- Conditional filtering enables analysts to quickly isolate relevant records.
- Operators such as `AND`, `OR`, `NOT`, and `LIKE` improve the precision of security investigations.
- Structured database queries support faster incident response and operational decision-making.
