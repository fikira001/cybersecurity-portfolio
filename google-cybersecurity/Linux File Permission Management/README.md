# Linux File Permission Management

## Overview

This project demonstrates the use of Linux commands to examine, modify, and manage file and directory permissions. The objective was to ensure users were granted only the permissions required for their roles while removing unauthorized access to sensitive resources.

## Scenario

As a security professional supporting a research team, I reviewed file and directory permissions to verify that access controls aligned with organizational security policies. Where permissions were overly permissive, I updated them using Linux authorization commands.

## Skills Demonstrated

- Linux File System Navigation
- File Permission Management
- Access Control
- Principle of Least Privilege
- Linux Security Administration
- Bash Command Line

## Commands Used

| Command | Purpose |
|---------|---------|
| `ls -la` | View files, directories, hidden files, and permissions |
| `chmod` | Modify file and directory permissions |
| `cd` | Navigate directories |
| `pwd` | Display the current working directory |

## Key Tasks

- Examined existing file and directory permissions.
- Interpreted Linux permission strings.
- Removed unnecessary write permissions.
- Updated permissions on hidden files.
- Restricted access to sensitive directories.
- Applied the Principle of Least Privilege.

## Key Takeaways

- File permissions are a critical component of Linux security.
- Proper access control reduces the risk of unauthorized modification and disclosure.
- Linux provides flexible permission management through the `chmod` command.
- Following the Principle of Least Privilege strengthens the organization's security posture.
